# PRD Plan: Heroku MIA Slack World Cup 2026 App

Date: 2026-06-13

## Objective

Create a comprehensive PRD for a Slack app that showcases Heroku Managed Inference Agents (MIA)
using the FIFA World Cup 2026 as its domain - providing AI-powered conversational features and
match broadcasting within Slack.

## Problem Statement

Heroku Managed Inference Agents is a powerful AI platform, but teams lack a tangible, engaging
reference app that demonstrates its capabilities in a real-world context. Meanwhile, the World
Cup 2026 (hosted in US/Mexico/Canada, June-July 2026) is a high-interest event that teams
naturally want to follow during their workday - but switching between Slack and sports apps
breaks flow.

This app solves both: it showcases Heroku MIA's AI capabilities while giving Slack users a native World Cup companion.

## Target Users

- Heroku internal teams and customers who use Slack
- Developers evaluating Heroku MIA for their own projects (showcase/demo value)
- Any Slack workspace that wants World Cup 2026 engagement during the tournament

## Proposed Solution

A Slack Bolt (Node.js) app deployed to Heroku that integrates Heroku MIA for intelligent, conversational World Cup features:

1. AI-Powered Chat (primary interface) - Users @mention the bot or DM it to ask questions about
   World Cup schedules, teams, groups, and more. Heroku MIA provides grounded,
   conversational responses. This is the main way users interact with the app.

2. Two Quick-Access Slash Commands - For the most common quick-glance queries:
   - `/worldcup2026 schedule` - show the next 3 upcoming matches
   - `/worldcup2026 groups` - show the group/bracket table

3. Match Broadcasting - The app proactively posts match cards to a designated channel ~15 minutes before kickoff.

## Key Features (Scope)

- AI conversational responses via @mention and DM (primary interface)
- `/worldcup2026 schedule` - next 3 upcoming matches
- `/worldcup2026 groups` - group/bracket table
- Scheduled match broadcasting to a channel

## Strategic Fit

- Demonstrates Heroku MIA in a relatable, engaging use case
- Shows Heroku platform capabilities (deployment, config vars, add-ons)
- Timely - World Cup 2026 is actively happening
- Low-risk showcase - no customer data, no production dependencies

## Data Approach

Static/mock JSON data representing a representative sample of the tournament:

- 24 teams across 6 groups
- ~52 matches (group stage + knockout)
- No live API dependency (self-contained demo)

## Technical Constraints

- Node.js (18+) on Heroku
- Slack Bolt framework with Socket Mode
- Heroku MIA for AI inference (OpenAI-compatible endpoint)
- In-memory store (no database - acceptable for showcase, resets on dyno restart)
- Must follow AGENTS.md security guardrails (PII masking, prompt defense, toxicity detection, audit trail)

## Architecture Diagram

A C4 System Context diagram (Level 1) will be created in DIAGRAMS.md showing:

- Slack User [Person] - Asks World Cup questions, receives match updates
- World Cup 2026 Slack App [Software System] - The system being built (in focus)
- Slack [External System] - Team messaging and collaboration platform
- Heroku Managed Inference Agents [External System] - AI inference and language model service

The diagram will illustrate:

- User interactions via slash commands, @mentions, and DMs
- WebSocket connection between Slack and the app (Socket Mode)
- HTTPS communication between the app and Heroku MIA for AI inference
- Response flow with Block Kit UI back to Slack

## Design Decisions

- AI-first approach - Most queries handled conversationally through Heroku MIA rather than rigid
  slash commands. This better showcases MIA's capabilities.
- Minimal commands - Only `/worldcup2026 schedule` and `/worldcup2026 groups` for quick-glance
  data that benefits from structured display.
- No interactive features - Predictions, polls, trivia deferred to future iteration. Focus on
  the core AI + broadcasting experience.
