# task-17: Live Card Elapsed Refresh

## Details

Date: 2026-06-17

Dependencies: task-16

Decisions: none

## Description

The broadcast card currently only updates when the score changes. During live
matches the elapsed minute in the header stays frozen between goals, making the
card look broken. Modify the differ to track elapsed time and emit a diff on
every poll cycle where the minute changes, then have the scheduler refresh the
card on elapsed change (not just score change). This keeps the header minute
ticking every ~15 seconds.

## Acceptance Criteria

- (Event-driven) When the elapsed minute changes between polls, the system
  shall update the broadcast card with the new elapsed value
- (Event-driven) When the score changes, the system shall continue to update
  the card as before (no regression)
- (Event-driven) When neither score nor elapsed changes between polls, the
  system shall not call chat.update (no unnecessary API calls)
- (Ubiquitous) The differ shall include `elapsedChanged` (boolean) and
  `elapsed` (number or null) in every diff entry
- (Ubiquitous) The scheduler's `handleDiffs` shall call `updateCard` when
  `scoreChanged` OR `elapsedChanged` is true

## Implementation Plan

1. In `src/live-data/differ.js`:
   - Add `previousElapsed` Map (fixtureId -> elapsed number)
   - In `diffAll`, extract `elapsed` from each cache entry's data
   - Detect elapsed change: `currentElapsed !== previousElapsed`
   - Include `elapsedChanged` and `elapsed` in the diff result object
   - Update the emit condition: `newEvents.length > 0 || scoreChanged || elapsedChanged`
   - Update `previousElapsed` in state after comparison
   - Clear `previousElapsed` in `clear()`

2. In `src/broadcast/scheduler.js`:
   - In `handleDiffs`, destructure `elapsedChanged` and `elapsed` from each diff
   - Change the `updateCard` condition from `if (scoreChanged)` to
     `if (scoreChanged || elapsedChanged)`
   - Pass the diff's `elapsed` value directly to `updateCard` instead of
     fetching from `getFixtureById(fixtureId)?.elapsed`

3. No changes to poller.js, mapper.js, or cache.js (elapsed already flows
   through from `mapLiveFixture` into cache entries)

## Test Strategy

Modify existing tests and add new cases:

In `src/test/live-data-differ.test.js`:

1. Elapsed changes with no new events and no score change -> emits diff with
   elapsedChanged: true
2. Elapsed unchanged with no events/score -> no diff emitted
3. Both elapsed and score change -> single diff with both flags true
4. First poll seeds elapsed (no diff emitted, same as existing restart guard)
5. Elapsed transitions from number to null (halftime) -> emits elapsedChanged
6. Elapsed stays null across consecutive polls -> no diff emitted
7. Elapsed transitions from null to number (second half start) -> emits diff

In `src/test/broadcast-live.test.js`:

1. handleDiffs calls updateCard when elapsedChanged is true and scoreChanged
   is false
2. handleDiffs does NOT call updateCard when both elapsedChanged and
   scoreChanged are false
3. handleDiffs passes elapsed value from diff to updateCard
