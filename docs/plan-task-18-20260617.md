# task-18: Broadcast Card Redesign

## Details

Date: 2026-06-17

Dependencies: task-17

Decisions: none

## Description

Replace the current 2-block `formatLiveCard` (header + section) with the new
3-block layout specified in FR-17/FR-18/TR-14. The redesign adds flag emojis,
full status text (no abbreviations), and a context block showing scorers
grouped by team. Also update `buildFixtureCard` (pre-match card) to use the
same section/context visual pattern for consistency.

## Acceptance Criteria

- (Ubiquitous) The live card shall use a 3-block layout: header block, section
  block, and context block
- (Ubiquitous) The header block shall display a status indicator emoji (green
  circle for live, red circle for finished) followed by full status text
  ("First Half", "Second Half", "Halftime", "Extra Time", "Match Finished")
  and elapsed minute where applicable
- (Ubiquitous) The section block shall display flag emoji + team name + score +
  team name + flag emoji using mrkdwn
- (Ubiquitous) The context block shall display scorers grouped by team (home
  scorers on left, away scorers on right, separated by a middle dot), each
  scorer prefixed with their team flag emoji and suffixed with minute
- (Event-driven) When no goals have been scored, the context block shall show
  "No goals yet"
- (Event-driven) When the card is rendered for a pre-match fixture, the section
  shall show team names with flags but no score, and the context shall show
  kickoff time
- (Ubiquitous) Flag emojis shall be resolved from `countries.json` via a
  `getFlag(teamId)` function using regional indicator symbol pairs
- (Ubiquitous) `countries.json` shall be the single source of truth for team
  flags, FIFA codes, and name aliases across the codebase

## Implementation Plan

1. Create `src/data/countries.json` - single source of truth for all 48 teams:
   - Array of objects with fields: `id`, `name`, `aliases` (optional), `flagISO`,
     `fifaCode`
   - `id` is the 3-letter internal code (e.g. "MEX", "USA", "RSA")
   - `flagISO` is ISO 3166-1 alpha-2 used for Unicode regional indicator lookup
   - `aliases` array handles API name variants (e.g. "Korea Republic" for KOR,
     "IR Iran" for IRN, "Bosnia and Herzegovina" for BOS)
   - 9 teams have aliases: BOS, CAP, CON, CZE, IRN, CIV, KOR, TUR, USA

2. Create `src/broadcast/flags.js` - flag emoji resolver from countries.json:
   - Require `../data/countries.json` at load time
   - Build a `TEAM_TO_ISO` Map from `country.id` -> `country.flagISO`
   - Internal `isoToFlag(iso)` converts 2-char ISO to regional indicator pair
     using `0x1F1E6 + charCode - 65`
   - Export only `getFlag(teamId)` - returns emoji or white flag fallback

3. Update `src/broadcast/format.js` - replace `formatLiveCard`:
   - Import `getFlag` from `./flags`
   - Keep signature as `formatLiveCard(fixture, score, elapsed)` - read
     `fixture.events` and `fixture.status` from the fixture object itself
     (getFixtureById already merges live cache data including these fields)
   - Use `fixture.status || 'Live'` directly (mapper already provides long-form
     status text  - no `formatStatusText` helper needed)
   - `isFinished` check via `statusText.startsWith('Match Finished')`
   - Block 1 (header plain_text, emoji: true):
     `{statusEmoji} {statusText} · {elapsed}'`
     - statusEmoji: green circle for live, red circle for finished
     - elapsed omitted when null (halftime, pre-match)
   - Block 2 (section mrkdwn):
     `{homeFlag} *{home}* {scoreHome} - {scoreAway} *{away}* {awayFlag}`
   - Block 3 (context block with single mrkdwn element): scorers line
     - Single text element to stay within Slack's 10-element context limit
     - Format: `{flag} Scorer min'  {flag} Scorer min' · {flag} Scorer min'`
     - Home scorers left of middle-dot separator, away scorers right
     - Empty state: "No goals yet"
   - Internal helper `formatScorers(events, fixture)` (NOT exported  - only used
     by formatLiveCard)

4. Update `src/broadcast/scheduler.js` - update `buildFixtureCard`:
   - Import `getFlag` from `./flags`
   - Redesign to 3-block layout for visual consistency:
     - Block 1 (header): "Fixture Starting Soon!" (unchanged)
     - Block 2 (section mrkdwn): `{homeFlag} *{home}* vs *{away}* {awayFlag}`
     - Block 3 (context mrkdwn): `Kickoff: {time}`

5. Refactor `scripts/sync-data.js` - use countries.json for FIFA_CODES:
   - Load `countries.json` via `loadJson()`
   - Build `FIFA_CODES` object by iterating countries: map `name.toLowerCase()`
     and each `alias.toLowerCase()` to `country.id`
   - Eliminates hardcoded FIFA code mapping  - countries.json is the single
     source of truth for both flags and team name resolution

6. Migrate existing `formatLiveCard` tests in `broadcast-format.test.js`:
   - Update test fixture objects to include `events` and `status` fields
   - Remove `formatStatusText` test (function no longer exists)
   - Existing assertions updated to match 3-block format

## Assumptions

- `countries.json` is the single source of truth for team ID, display name,
  ISO flag code, FIFA code, and API name aliases.
- Slack renders Unicode flag emojis across all modern clients (confirmed in PRD
  constraints).
- The `events` array from live cache contains `teamId` field on each event,
  allowing scorer grouping by home/away team.
- The `status` field from live cache already arrives as long-form text from the
  mapper (e.g. "First Half", "Match Finished")  - no abbreviation expansion
  needed at format time.
