# task-2: Bolt App Bootstrap

## Details

Date: 2026-06-15

Dependencies: task-1

Decisions: none

## Description

Bootstrap the Slack Bolt application with Socket Mode connectivity. This replaces
the placeholder scaffolding from task-1 with a real `@slack/bolt` app instance
that connects to Slack over WebSocket (Socket Mode). No slash commands or event
handlers are registered -- those come in later tasks. This task establishes the
foundational app structure that all subsequent features build upon.

Addresses requirements TR-1 (Node.js with @slack/bolt) and TR-2 (Socket Mode
connection).

## Acceptance Criteria

- The system shall have `@slack/bolt` listed as a production dependency in
  `package.json`
- The system shall create a Bolt `App` instance configured for Socket Mode
  (`socketMode: true`)
- The system shall read `SLACK_BOT_TOKEN`, `SLACK_SIGNING_SECRET`, and
  `SLACK_APP_TOKEN` from environment variables
- When `npm start` is run without environment variables, the system shall fail
  gracefully with a clear error message listing missing variables
- The system shall export the Bolt `app` instance from `src/app.js` for
  testability
- When `npm test` is run, the system shall pass with at least one meaningful
  test verifying the app module structure
- When `npm run lint` is run, the system shall pass with no errors
- The system shall not contain any secrets or credentials in committed code

## Implementation Plan

1. Install @slack/bolt dependency:

   ```bash
   npm install @slack/bolt
   ```

   This adds `@slack/bolt` to `package.json` under `dependencies`.

2. Create `src/app.js` - Separates app construction from startup for testability:

   ```javascript
   'use strict';

   const { App } = require('@slack/bolt');

   const app = new App({
     token: process.env.SLACK_BOT_TOKEN,
     signingSecret: process.env.SLACK_SIGNING_SECRET,
     appToken: process.env.SLACK_APP_TOKEN,
     socketMode: true,
   });

   module.exports = { app };
   ```

3. Replace `src/index.js` - Entry point with env validation:

   ```javascript
   'use strict';

   const REQUIRED_ENV_VARS = ['SLACK_BOT_TOKEN', 'SLACK_SIGNING_SECRET', 'SLACK_APP_TOKEN'];

   function validateEnv() {
     const missing = REQUIRED_ENV_VARS.filter((key) => !process.env[key]);
     if (missing.length > 0) {
       throw new Error(
         `Missing required environment variables: ${missing.join(', ')}. ` +
           'Set these in your Heroku config vars or local .env file.'
       );
     }
   }

   async function start() {
     validateEnv();

     const { app } = require('./app');

     await app.start();
     console.log('World Cup 2026 Slack app is running (Socket Mode)');
   }

   start().catch((err) => {
     console.error('Failed to start app:', err.message);
     process.exit(1);
   });
   ```

4. Create `.env.example` - Document required env vars:

   ```sh
   # Required Slack credentials (never commit actual values)
   SLACK_BOT_TOKEN=xoxb-your-bot-token
   SLACK_SIGNING_SECRET=your-signing-secret
   SLACK_APP_TOKEN=xapp-your-app-level-token
   ```

5. Update `.gitignore` - Add `.env.local` for extra safety:

   ```text
   node_modules
   .env
   .env.local
   ```

6. Rewrite `src/test/index.test.js` - Test app module structure:

   ```javascript
   'use strict';

   process.env.SLACK_BOT_TOKEN = 'xoxb-test-token';
   process.env.SLACK_SIGNING_SECRET = 'test-signing-secret';
   process.env.SLACK_APP_TOKEN = 'xapp-test-app-token';

   const { App } = require('@slack/bolt');
   const { app } = require('../app');

   describe('app module', () => {
     test('exports an app instance', () => {
       expect(app).toBeDefined();
     });

     test('app is an instance of Bolt App', () => {
       expect(app).toBeInstanceOf(App);
     });

     test('app is configured for socket mode', () => {
       expect(app.receiver).toBeDefined();
       expect(app.receiver.constructor.name).toBe('SocketModeReceiver');
     });
   });
   ```

7. Add `Procfile` (new file, project root) - Heroku process declaration:

   ```text
   worker: node src/index.js
   ```

   Uses `worker` type to avoid R10 timeout (Socket Mode doesn't bind HTTP
   port).

## Test Strategy

Run `npm test` and `npm run lint` before submitting for review.

Unit tests (automated):

- Verify `require('./app')` returns `{ app }` where `app` is an instance of Bolt
  `App` class
- Verify the app's receiver is a `SocketModeReceiver` (confirms Socket Mode
  enabled)
- Verify the app has expected structure

Manual verification:

- Run `npm start` without env vars - expect clear error listing missing
  variables
- Run with valid credentials - expect "World Cup 2026 Slack app is running
  (Socket Mode)" message
- Verify no secrets in committed code
