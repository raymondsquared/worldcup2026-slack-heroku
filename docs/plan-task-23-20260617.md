# Plan: Task-23 - Source Citation in AI Responses

Date: 2026-06-17
Requirement: FR-12
Depends on: task-22 (Post-LLM Fallback Integration)

## Goal

When the fallback path fires and web search results are used, MIA must cite
its sources in the answer. Format: "According to [source name] (URL)..." -
only added when web context is present (not on local-only responses).

## Current State

The retry path in `src/mia/index.js` (lines 71-89) builds augmentedMessages
with searchContext appended to the system prompt. The search context already
includes numbered sources:

```
[1] ESPN Report
Latest match analysis for USA vs Mexico.
Source: https://espn.com/article
```

MIA receives this but has no instruction to cite the sources in its answer.

## Design

### Approach: Reference instruction appended only on retry path

Add a `REFERENCE_INSTRUCTION` constant in `src/mia/personas.js`. This
instruction tells MIA to attribute claims to the numbered web sources using
"According to [source name] (URL)" format. It is injected ONLY in the
augmented retry messages - never in the initial call or local-only responses.

Why this works:
- Clean separation: BASE_INSTRUCTION handles XML format + confidence scoring;
  REFERENCE_INSTRUCTION handles attribution (only relevant when web data exists)
- No change to the initial call path - citations only appear when web search
  actually provided results
- Claude Opus reliably follows layered prompt instructions
- The numbered format `[1] Title ... Source: URL` in the context gives MIA
  clear references to cite

### Reference Instruction Wording

```
When citing information from web search results, attribute the source inline
using this format: "According to [source name] (URL)". Use the source title
and URL from the numbered results above. Every factual claim from web results
must have at least one citation. Do not invent sources.
```

### Changes

1. `src/mia/personas.js` - Add REFERENCE_INSTRUCTION constant, export it
2. `src/mia/index.js` - Import REFERENCE_INSTRUCTION; append it to system
   prompt in augmentedMessages (after searchContext)
3. `src/test/mia.test.js` - Add tests verifying:
   - Reference instruction present in retry messages when web context exists
   - Reference instruction NOT present in initial call messages
   - Reference instruction NOT present when webSearch returns null

## Implementation Detail

### personas.js addition

```javascript
const REFERENCE_INSTRUCTION =
  'When citing information from web search results, attribute the source' +
  ' inline using this format: "According to [source name] (URL)".' +
  ' Use the source title and URL from the web search results provided.' +
  ' Every factual claim from web results must have at least one citation.' +
  ' Do not invent sources.' +
  ' If you cannot provide a confident answer even with the web results,' +
  ` set <answer> to exactly: "${LOW_CONFIDENCE_PHRASE}"`;
```

Export alongside existing constants.

### index.js change (lines 75-82)

Current:
```javascript
const augmentedMessages = [
  {
    role: 'system',
    content: systemPrompt + (grounded ? `\n\nContext:\n---\n${grounded}\n---` : '')
      + `\n\n${searchContext}`,
  },
  { role: 'user', content: sanitized },
];
```

After:
```javascript
const augmentedMessages = [
  {
    role: 'system',
    content: systemPrompt + (grounded ? `\n\nContext:\n---\n${grounded}\n---` : '')
      + `\n\n${searchContext}`
      + `\n\n${REFERENCE_INSTRUCTION}`,
  },
  { role: 'user', content: sanitized },
];
```

REFERENCE_INSTRUCTION goes AFTER searchContext so the LLM sees the sources
first, then the instruction to cite them.

## Test Plan

### New tests in src/test/mia.test.js

1. "includes reference instruction in retry messages when web search succeeds"
   - Mock chat to return low-confidence XML on first call
   - Mock webSearch to return context
   - Mock chat again for retry
   - Assert the retry system message contains REFERENCE_INSTRUCTION text

2. "does not include reference instruction in initial call"
   - Mock chat to return a confident answer
   - Assert chat was called once with messages NOT containing citation text

3. "does not include reference instruction when web search returns null"
   - Mock chat to return low-confidence
   - Mock webSearch to return null
   - Assert chat was called once (no retry, so no reference instruction)

4. "cited response with URLs passes through pipeline unmolested"
   - Mock chat to return low-confidence on first call
   - Mock webSearch to return context
   - Mock retry to return XML with answer containing
     "According to ESPN (https://espn.com), USA won 2-1"
   - Assert final output includes the full citation string with URL
   - Verifies PII masking and toxic filter do not strip citation URLs

### Existing tests unaffected

- parseResponse tests: no change to XML parsing
- Low-confidence detection tests: no change to detection logic
- Web search tests: no change to search/sanitize logic
- Persona tests: REFERENCE_INSTRUCTION is separate from BASE_INSTRUCTION

## Risks

- Low: Reference instruction adds ~50 tokens to retry prompt. Well within
  context limits.
- Low: MIA may not always format citations perfectly. Acceptable - the
  instruction guides it, and Claude Opus is reliable at following format
  instructions.
- None: No new dependencies. No env vars. No API changes.
