# Task 25: Playlist API Client

## Details

Date: 2026-06-19

Dependencies: none

Decisions: none

## Description

Build a client module (`src/highlights/client.js`) that fetches playlist items
from a configured video playlist API. The client handles pagination, timeout,
sanitization of untrusted title data, and graceful degradation when env vars
are missing. This is the data ingestion layer for the daily highlights feature.

## Acceptance Criteria

- (Ubiquitous) The system shall export a `fetchPlaylistItems()` function that
  returns an array of `{ videoId, title, publishedAt, url }` objects
- (Event-driven) When `HIGHLIGHTS_API_KEY` or `HIGHLIGHTS_PLAYLIST_ID` env
  vars are not set, the system shall return an empty array and log a warning
  once at module load
- (Event-driven) When the API returns paginated results (nextPageToken), the
  system shall follow all pages until the full playlist is retrieved
- (Event-driven) When a request exceeds the configured timeout (default 10s),
  the system shall abort and throw an error
- (Unwanted) If the API returns an error status, the system shall throw with
  the HTTP status code but never include the API key in the error message
- (Event-driven) When items are fetched successfully, the system shall apply
  `sanitizeInput()` to each video title before returning (treats playlist
  data as untrusted)

## Implementation Plan

1. Create `src/highlights/` directory
2. Create `src/highlights/client.js` with:
   - Module-level constants: read `HIGHLIGHTS_API_KEY` and
     `HIGHLIGHTS_PLAYLIST_ID` from env at load time
   - `MAX_PAGES = 10` safety cap (playlist ~100 videos = 2 pages at 50/page;
     cap prevents runaway loops on malformed API responses)
   - API base URL:
     `https://www.googleapis.com/youtube/v3/playlistItems`
     Query params: `part=snippet&playlistId={id}&key={key}&maxResults=50&pageToken={token}`
   - Log warning once if either env var is missing
   - `fetchPlaylistItems()` async function:
     a. Return `[]` if env vars missing
     b. Loop (max MAX_PAGES iterations): fetch endpoint with pageToken param
     c. Use `AbortSignal.timeout()` for configurable timeout
     d. Parse JSON response, handle empty `items` array (valid, continue)
     e. For each item: skip if `snippet.resourceId.videoId` is missing;
        use empty string for missing `title`; skip if `publishedAt` missing
     f. Apply `sanitizeInput()` to each title
     g. Build video URL: `https://www.youtube.com/watch?v={videoId}`
     h. Follow nextPageToken until exhausted or MAX_PAGES reached
     i. Return full array of items
3. Error handling:
   - Non-2xx response: throw with status code (no API key in message)
   - Network/timeout: let native fetch error propagate
   - Malformed items: skip silently (log count of skipped at end)
4. Create test file `src/test/highlights-client.test.js`

## Test Strategy

Run `npm test` and `npm run lint`.

- Mock `global.fetch` to simulate API responses
- Test single page response (no nextPageToken): returns items correctly
- Test multi-page response: follows pagination, returns all items
- Test error response (4xx/5xx): throws with status, no API key leaked
- Test timeout: AbortSignal fires, error propagates
- Test missing env vars: returns empty array, no fetch called
- Test sanitization: titles with injection markers are cleaned
- Test URL construction: videoId mapped to correct URL format
- Test max page cap: stops at MAX_PAGES even if nextPageToken present
- Test malformed items: missing videoId skipped, missing title defaults empty
