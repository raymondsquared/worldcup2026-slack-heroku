# Task 27: Fixture Data Enrichment

## Details

Date: 2026-06-19

Dependencies: task-26

Decisions: none

## Description

Build an enrichment module (`src/highlights/enrich.js`) that persists
`highlightsURL` into `fixtures.json` for matched fixtures. Uses atomic file
writes (temp file with PID-based naming + rename) and a module-level mutex
(promise queue) to serialize concurrent writes within the same process.

## Acceptance Criteria

- (Ubiquitous) The system shall export an `enrichFixtures(matches)` function
  that writes `highlightsURL` to matched fixture entries in `fixtures.json`
- (Event-driven) When matches array is empty, the system shall skip the
  write entirely and return 0
- (Event-driven) When a fixture already has a `highlightsURL` field, the
  system shall overwrite it with the new value (upsert)
- (Event-driven) When a fixture does not have a `highlightsURL` field, the
  system shall add it
- (Ubiquitous) The system shall write to a temp file
  (`fixtures.tmp.${process.pid}.json`) then rename over the original
  (atomic write)
- (State-driven) While a write is in progress, the system shall queue
  subsequent write requests and execute them serially (mutex)
- (Ubiquitous) The system shall return the count of fixtures updated

## Implementation Plan

1. Create `src/highlights/enrich.js`
2. Module-level mutex using a promise chain:
   - `let writeQueue = Promise.resolve()`
   - Each call chains onto writeQueue via
     `writeQueue = writeQueue.then(() => doWrite(matches))`
   - Wrap doWrite in try/finally so the queue always advances even on
     failure (prevents deadlock)
3. `enrichFixtures(matches)` inner `doWrite` function (runs inside mutex):
   a. If matches is empty or null, return 0
   b. Read `src/data/fixtures.json` from disk (fresh read INSIDE mutex
      to prevent stale reads from concurrent calls)
   c. For each match: find fixture by ID in the array
      - If fixture not found: log warning, skip (do not throw)
      - If found: set `fixture.highlightsURL = match.url`
   d. Write JSON to temp file using `JSON.stringify(data, null, 2)`
      (preserves existing 2-space indentation) at path
      `src/data/fixtures.tmp.${process.pid}.json`
   e. Rename temp file over `src/data/fixtures.json` (atomic)
   f. Return count of fixtures that were updated
4. Error handling:
   - If read fails: throw (caller handles; queue still advances via finally)
   - If write/rename fails: attempt to clean up temp file, then throw
   - Queue never deadlocks: try/finally ensures next queued call proceeds
5. Create test file `src/test/highlights-enrich.test.js`

## Test Strategy

Run `npm test` and `npm run lint`.

- Test add new highlightsURL field to fixture
- Test overwrite existing highlightsURL field
- Test empty matches: no file write, returns 0
- Test atomic write: verify temp file created then renamed (fs spy)
- Test concurrent writes: fire 2 enrichFixtures in parallel, verify
  serialized execution via write order
- Test fixture not found in JSON: skip gracefully, still update others
- Test write failure: temp file cleaned up
