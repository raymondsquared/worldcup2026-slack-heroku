# Task 29: Daily Highlight Digest (AI + Post)

## Details

Date: 2026-06-19

Dependencies: task-27

Decisions: none

## Description

Build `src/highlights/digest.js` that generates an AI-written daily digest
from matched highlight videos and posts it to the broadcast channel. Uses
the existing `ask()` function in recap mode with a digest-specific system
prompt. Sanitizes video titles before MIA context. Logs match ratio and
warns if below 0.9 threshold.

## Acceptance Criteria

- (Ubiquitous) The system shall export a `generateDigest(client, matches)`
  function that generates and posts a daily highlight digest
- (Event-driven) When matches array is empty, the system shall skip digest
  generation and posting, log a skip message, and return null
- (Ubiquitous) The system shall sanitize all video titles via
  `sanitizeInput()` before including them in MIA context
- (Ubiquitous) The system shall use `ask()` in recap mode with a custom
  system prompt for digest generation
- (Ubiquitous) The system shall post the digest to BROADCAST_CHANNEL_ID as
  a standalone message (not threaded)
- (Event-driven) When MIA returns null (toxic filter), the system shall
  log a warning and return null without posting
- (Ubiquitous) The system shall log match ratio stats: total highlight
  videos from previous day, matched count, unmatched count
- (Event-driven) When matched/total ratio is below 0.9, the system shall
  emit a console.warn alert

## Implementation Plan

1. Create `src/highlights/digest.js`
2. Constants:
   - `CHANNEL = process.env.BROADCAST_CHANNEL_ID`
3. `buildDigestContext(matches)` function:
   a. For each match: get fixture data via `getFixtureById(match.fixture.id)`
   b. Build structured context with team names, scores, and highlight URLs
   c. Sanitize video titles with `sanitizeInput()` before inclusion
   d. Format with `---` delimiters (same pattern as recap.js)
   e. Return context string
4. `DIGEST_SYSTEM_PROMPT` constant:
   - Instructs MIA to write a 2-4 paragraph daily highlight digest
   - Include all highlight links naturally in the text
   - Tone: engaging, concise, football-focused
   - No XML format needed (recap mode returns plain text)
5. `logMatchStats(totalHighlights, matchedCount)` function:
   a. Calculate unmatched = totalHighlights - matchedCount
   b. Calculate ratio = matchedCount / totalHighlights (or 1.0 if total is 0)
   c. Log: total, matched, unmatched, ratio
   d. If ratio < 0.9, emit console.warn
6. `generateDigest(client, matches, stats)` function:
   a. Call `logMatchStats(stats.totalHighlights, matches.length)`
   b. If matches is empty or null: log skip message, return null
   c. Build context via `buildDigestContext(matches)`
   d. Call `ask(userPrompt, { context, recap: true, systemOverride })`
   e. If result is null (toxic filter): log warning, return null
   f. Post to BROADCAST_CHANNEL_ID via `client.chat.postMessage()`
   g. Log success, return the digest text
7. Create test file `src/test/highlights-digest.test.js`

## Test Strategy

Run `npm test` and `npm run lint`.

- Test successful digest generation and posting (mock ask + client)
- Test empty matches: no ask() call, no post, log skip message, returns null
- Test MIA returns null (toxic): no post, log warning, returns null
- Test context building: verify sanitizeInput applied to titles, verify
  fixture data included (team names, scores, URLs)
- Test match ratio logging: above threshold (info only), below threshold
  (console.warn emitted)
- Test Slack post failure: error propagates to caller
