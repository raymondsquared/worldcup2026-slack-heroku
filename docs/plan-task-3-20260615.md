# task-3: Heroku Infrastructure Setup

## Details

Date: 2026-06-15

Dependencies: task-2

Decisions: none

## Description

Configure the Heroku deployment infrastructure so the app is deployable via
`git push heroku main` and runs successfully as a worker dyno. The Procfile
already exists from task-2; this task adds `app.json` for Heroku platform
metadata, documents all required config vars, and verifies end-to-end
deployment.

Addresses requirements TR-12 (deployable via git push with Procfile and config
vars) and C-5 (secrets stored in Heroku config vars, never in code).

## Acceptance Criteria

- The system shall include an `app.json` at the project root declaring the app
  name, description, Node.js buildpack, required environment variables, and
  dyno formation
- The system shall declare `SLACK_BOT_TOKEN`, `SLACK_SIGNING_SECRET`, and
  `SLACK_APP_TOKEN` as required config vars in `app.json`
- The system shall specify the `worker` formation in `app.json` matching the
  existing Procfile
- The system shall rely on the `engines.node` field in `package.json`
  (>=22.0.0) for Heroku's Node.js version selection (Heroku reads this from
  `package.json`, not `app.json`)
- When deployed to Heroku via `git push heroku main`, the system shall build
  successfully and start the worker dyno without errors
- The system shall not contain any secrets or credentials in committed code
- When `npm run lint` is run, the system shall pass with no errors

## Implementation Plan

1. Create `app.json` at the project root with Heroku platform metadata:

   ```json
   {
     "name": "rb-heroku-mia-slack-worldcup26",
     "description": "Slack app showcasing Heroku Managed Inference Agents with World Cup 2026 features",
     "keywords": ["slack", "heroku", "world-cup", "ai"],
     "repository": "https://github.com/heroku/rb-heroku-mia-slack-worldcup2026",
     "stack": "heroku-24",
     "buildpacks": [
       { "url": "heroku/nodejs" }
     ],
     "env": {
       "SLACK_BOT_TOKEN": {
         "description": "Slack Bot User OAuth Token (xoxb-...)",
         "required": true
       },
       "SLACK_SIGNING_SECRET": {
         "description": "Slack app signing secret for request verification",
         "required": true
       },
       "SLACK_APP_TOKEN": {
         "description": "Slack App-Level Token for Socket Mode (xapp-...)",
         "required": true
       }
     },
     "formation": {
       "web": {
         "quantity": 0
       },
       "worker": {
         "quantity": 1,
         "size": "private-s"
       }
     }
   }
   ```

2. Create `infrastructure/README.md` documenting deployment steps:

   - Prerequisites (Heroku CLI, Git remote)
   - How to set config vars (`heroku config:set`)
   - How to deploy (`git push heroku main`)
   - How to verify (`heroku logs --tail`, `heroku ps`)
   - How to scale the worker dyno

3. Verify existing Procfile is correct (`worker: node src/index.js`) -- no
   changes needed.

4. Verify `.gitignore` excludes `.env` and `.env.local` -- no changes needed
   (already in place from task-2).

5. Run `npm run lint` to confirm no markdown or JS lint errors.

6. Deploy to Heroku and verify:

   ```bash
   git push heroku main
   heroku ps
   heroku logs --tail
   ```

   Confirm the worker dyno starts and the app logs "World Cup 2026 Slack app
   is running (Socket Mode)".

## Test Strategy

Run `npm test` and `npm run lint` before submitting for review.

Automated:

- Validate `app.json` is well-formed JSON (parse without error)
- Verify `app.json` env keys match `REQUIRED_ENV_VARS` in `src/index.js`

Manual:

- Deploy to Heroku via `git push heroku main`
- Confirm build succeeds with Node.js buildpack
- Confirm `heroku ps` shows worker dyno in `up` state
- Confirm `heroku logs --tail` shows the startup success message
- Confirm no secrets are in committed code (`grep -r xoxb . --include='*.js'`
  returns nothing)
