# Task 30: Daily Scheduler

## Details

Date: 2026-06-19

Dependencies: task-29

Decisions: ADR-4 (In-Process Clock via setInterval)

## Description

Build `src/highlights/scheduler.js` that orchestrates the full daily
highlights job: fetch playlist -> filter previous day -> match -> enrich ->
digest -> post. Uses setInterval with a time-of-day check, persists
`lastRunDate` to prevent duplicate runs, and includes missed-run recovery
on startup.

## Acceptance Criteria

- (Ubiquitous) The system shall export `start(client)` and `stop()`
  functions for lifecycle management
- (Ubiquitous) The system shall check every 30 minutes whether the current
  UTC hour matches `HIGHLIGHTS_RUN_HOUR_IN_UTC` (default 12)
- (Event-driven) When the run hour matches and today has not been run, the
  system shall execute the full highlights pipeline
- (Event-driven) When the pipeline completes (digest posted or zero
  highlights), the system shall persist today's date to
  `src/data/.last-highlights-run`
- (Event-driven) When the pipeline fails (digest post error), the system
  shall NOT update lastRunDate (retry on next tick)
- (Event-driven) On startup, if today's date > lastRunDate AND current UTC
  hour >= run hour, the system shall fire immediately (missed-run recovery)
- (Event-driven) On startup, if today's date > lastRunDate AND current UTC
  hour < run hour, the system shall wait for the normal tick
- (Ubiquitous) The system shall gracefully no-op if HIGHLIGHTS_API_KEY is
  not set (log warning once, skip)

## Implementation Plan

1. Create `src/highlights/scheduler.js`
2. Constants:
   - `RUN_HOUR = Number(process.env.HIGHLIGHTS_RUN_HOUR_IN_UTC) || 12`
   - `CHECK_INTERVAL_IN_MS = 30 * 60_000` (30 minutes)
   - `LAST_RUN_PATH = path.join(__dirname, '..', 'data', '.last-highlights-run')`
3. Helper functions:
   - `readLastRunDate()`: read file, return date string or null
   - `writeLastRunDate(dateStr)`: write date string to file
   - `getYesterdayUTC()`: return YYYY-MM-DD for previous UTC day
   - `getTodayUTC()`: return YYYY-MM-DD for current UTC day
4. `run(client)` - the full pipeline:
   a. Fetch playlist via `fetchPlaylistItems()`
   b. Filter videos published on previous UTC day (compare publishedAt date)
   c. Load fixtures from disk (`fixtures.json`)
   d. Match via `matchVideosToFixtures(videos, fixtures)`
   e. Enrich via `enrichFixtures(matches)`
   f. Generate + post digest via `generateDigest(client, matches, stats)`
   g. Write lastRunDate on success (or zero highlights)
   h. Wrap in try/catch: log error, do NOT write lastRunDate on failure
5. `tick(client)` - called every minute:
   a. If current UTC hour !== RUN_HOUR, return
   b. If `readLastRunDate() === getTodayUTC()`, return (already ran)
   c. Call `run(client)`
6. `start(client)`:
   a. If !HIGHLIGHTS_API_KEY: log warning, return (graceful no-op)
   b. Check for missed run: if lastRunDate < today AND hour >= RUN_HOUR,
      call `run(client)` immediately
   c. Start interval: `setInterval(() => tick(client), CHECK_INTERVAL_IN_MS)`
7. `stop()`: clear interval
8. Wire into `src/index.js`:
   - `const highlightsScheduler = require('./highlights/scheduler')`
   - Call `highlightsScheduler.start(app.client)` after `scheduler.start()`
9. Create test file `src/test/highlights-scheduler.test.js`

## Test Strategy

Run `npm test` and `npm run lint`.

- Test tick at run hour with no prior run: triggers pipeline
- Test tick at run hour with today already run: skips
- Test tick at non-run hour: skips
- Test missed-run recovery on start: fires immediately
- Test start when hour < run hour and lastRunDate is stale: does NOT fire
- Test start without HIGHLIGHTS_API_KEY: logs warning, no interval started
- Test pipeline failure: lastRunDate NOT written, retry on next tick
- Test pipeline success: lastRunDate written
- Test previous-day video filter logic
