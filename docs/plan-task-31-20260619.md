# Task 31: `/worldcup2026 highlights` Command

## Details

Date: 2026-06-19

Dependencies: task-29

Decisions: none

## Description

Add a `highlights` subcommand to the `/worldcup2026` slash command that renders
recent matched highlights on demand as rich Block Kit cards, reusing the
daily-digest card layout (`buildMatchBlocks`). This task covers the command, its
discovery surfaces (manifest/diagram/help text), and the final selection
behavior (latest-N across all days).

(Originally split across three working tasks - the command, a docs/manifest
sync, and a behavior fix - now consolidated into one since they ship together as
a single feature.)

## Acceptance Criteria

- (Ubiquitous) `/worldcup2026 highlights` shall respond with recent matched
  highlights as Block Kit cards: flags, score, stage/group, kickoff time, and a
  "Watch" link button, reusing `buildMatchBlocks`.
- (Ubiquitous) The command shall surface the most recent N fixtures that have a
  `highlightsURL`, ordered newest-first across all match days (not limited to
  the previous calendar day).
- (Ubiquitous) N shall default to 5 and be overridable via
  `HIGHLIGHTS_COMMAND_FIXTURES_LIMIT`; an invalid value falls back to 5.
- (Event-driven) When no fixture has a `highlightsURL`, the command shall show a
  friendly "No highlights available yet" message.
- (Ubiquitous) Fixtures missing `highlightsURL`, `dateAndTimeInUTC`, or `teams`
  shall be ignored.
- (Ubiquitous) All command discovery surfaces shall list `highlights`: the Slack
  app manifest `description`/`usage_hint`, the architecture diagram, and the
  in-command usage text.
- (Ubiquitous) Tests shall cover: latest-N ordering, env/arg limit override,
  invalid fallback, exclusions, empty state, and the help-text update.

## Implementation

### Command + query

- `src/highlights/query.js`: `getLatestHighlights(limit)` reads `fixtures.json`,
  filters to fixtures with a `highlightsURL` (and a date + teams), sorts by
  `dateAndTimeInUTC` descending, slices to the limit, and maps to
  `{ fixture, url, title }`. Limit resolution: explicit arg >
  `HIGHLIGHTS_COMMAND_FIXTURES_LIMIT` > `DEFAULT_LIMIT` (5).
- `src/commands/worldcup.js`: register a `highlights` case calling
  `getLatestHighlights()`, build cards via `buildMatchBlocks` under a static
  "Recent Highlights" header (results may span days), respond ephemerally;
  empty-state message when none.
- `.env.example`: document `HIGHLIGHTS_COMMAND_FIXTURES_LIMIT=5`.

### Discovery surfaces

- `docs/slack-app-manifest.yaml`: `description` and `usage_hint` for
  `/worldcup2026` include `highlights` (`[schedule | groups | highlights]`).
  Only the manifest change affects what users see in Slack; the app must be
  re-installed for the new hint to take effect.
- `DIAGRAMS.md`: the Slash Commands C4 component lists `highlights` alongside
  `schedule` and `groups`.
- In-command usage text already lists `highlights`.

### Tests

- `src/test/highlights-query.test.js`: latest-N ordering, env override,
  explicit-arg override, invalid fallback, exclusions, empty.
- `src/test/commands.test.js`: highlights subcommand block shape, header text,
  empty state, help-text update.

## Technical Notes

- Reuses `buildMatchBlocks` from `src/highlights/digest.js` - single source of
  truth for the card layout. No AI generation - instant response, no MIA latency.
- Command response is ephemeral (default for `respond()`) so only the requesting
  user sees it.
- Command-only behavior: the daily digest/scheduler (task-29/30) is unaffected -
  it still posts for the previous day on its schedule.
- Data caveat: only fixtures with a synced `highlightsURL` appear. If results
  look sparse, run `npm run sync:highlights` to backfill (task-28 behavior).
