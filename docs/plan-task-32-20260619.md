# Task 32: Fix Flaky Enrich/Poller Test Race

## Details

Date: 2026-06-19

Dependencies: task-27

Decisions: none

## Description

The full Jest suite failed intermittently (~1 run in 9) with:

```text
SyntaxError: Unexpected end of JSON input
  at JSON.parse  ->  require('../data/fixtures.json')   [mapper.js:3, via poller.js:4]
  at src/test/live-data-poller.test.js:70
```

## Root Cause

A cross-worker file race on the committed `src/data/fixtures.json`:

- `highlights-enrich.test.js` wrote the real `fixtures.json` via non-atomic
  `fs.writeFileSync` (truncate + write) in `beforeAll`/`afterAll`/`beforeEach`.
- `live-data-poller.test.js` calls `jest.resetModules()` in `beforeEach`, then
  re-`require`s `poller.js` -> `mapper.js` -> `require('../data/fixtures.json')`
  on every test, forcing a fresh disk read.
- Jest runs suites in parallel workers (no `maxWorkers` cap). When the poller's
  re-require landed inside the enrich suite's truncate window, it read an empty
  file -> `Unexpected end of JSON input`.

Not a production bug: `enrich.js` already writes atomically (temp + rename) at
runtime. The flake was purely the test mutating shared source data.

## Fix

Stop the enrich test from touching the committed file:

- `src/highlights/enrich.js`: resolve `FIXTURES_PATH` from
  `process.env.HIGHLIGHTS_FIXTURES_PATH`, defaulting to the real
  `src/data/fixtures.json`. Pure seam - production behavior unchanged.
- `src/test/highlights-enrich.test.js`: create a throwaway copy under
  `os.tmpdir()` via `fs.mkdtempSync`, point `HIGHLIGHTS_FIXTURES_PATH` at it
  before requiring enrich, seed/reset it from the real file, and remove the
  temp dir in `afterAll`. The real `fixtures.json` is never written.

## Verification

- `git status src/data/fixtures.json` -> clean after a test run (no longer
  mutated)
- `npx jest src/test/highlights-enrich.test.js` -> 8/8 pass against temp copy
- Full suite run 50x consecutively -> 0 failures (previously ~1 in 9). At the
  old rate the chance of 50 clean runs by luck is ~0.3%, confirming the fix.
