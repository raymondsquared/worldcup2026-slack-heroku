# Task 33: Broadcast & Command UI Polish

## Details

Date: 2026-06-19

Dependencies: task-31

Decisions: none

## Description

A cluster of UI/UX fixes and improvements to the Slack-facing surfaces, found
while exercising the live broadcast and `/worldcup2026` commands:

1. Live match card froze mid-match (stuck at `First Half · 16' · 0-0`) while
   thread events kept posting.
2. `/worldcup2026 schedule` and `groups` rendered as cramped, unsorted walls of
   text - no flags, no standings, groups in insertion order.
3. Match kickoff time was only on the schedule; highlights cards showed no time.

All display-layer only; no data model or sync changes.

## Part 1: Fix frozen live card (bug)

Root cause: `formatScorers` (`src/broadcast/format.js`) returned an empty
string when goals existed but none matched `homeTeamId`/`awayTeamId` (e.g. live
API team id unmapped -> `teamId` null). An empty `context` element is rejected
by Slack (`invalid_blocks`), so `chat.update` threw and `updateCard` swallowed
it - freezing the card. Thread events kept posting because `postEventAlert` uses
different formatters that never call `formatScorers`.

- `formatScorers`: never return `''`; fall back to `'No goals yet'` when no goal
  matches either team.
- `updateCard` (`src/broadcast/scheduler.js`): log the Slack error code
  (e.g. `invalid_blocks`) instead of swallowing, so future block bugs surface.
- Tests: goals with `teamId` null / foreign id / mixed matched+unmatched.

## Part 2: Beautify schedule & groups

In `src/commands/worldcup.js`, reusing `getFlag` and the header/divider pattern:

- schedule (`upcomingBlocks` + `liveBlocks`): flags on both teams, kickoff/status
  moved to a greyed `context` line, emoji headers (`📅`, `🟢`), dividers.
- groups (`groupsBlocks`): sort groups A->L (was insertion order, Group J first),
  rank-sort teams within each group, render a mini standings line per team:
  `` `1.` 🇲🇽 Mexico - 3 pts (+2) `` with `pt`/`pts` pluralization and `+N`/`-N`/
  `±0` goal-difference. Dividers between groups. Team names NOT bold (per
  request). Standings data (`rank`/`points`/`goalsDiff`) is real - synced via
  `fetchStandings` in `scripts/sync-data.js`.
- UK constituent-country flags: England and Scotland both carry `flagISO: GB`
  (both rendered as 🇬🇧). Added a team-id override map in
  `src/broadcast/flags.js` returning Slack named emoji `:flag-england:` /
  `:flag-scotland:` (and `:flag-wales:` for completeness). These render in
  mrkdwn, where all flag usages live. Applies everywhere `getFlag` is used
  (schedule, groups, highlights, live + pre-match broadcast cards).

## Part 3: Post-match recap as standalone card

The AI-generated recap was posted as a thread reply under the live card, making
it invisible to users who hadn't already opened the thread. Move it to a
standalone channel message with a rich Block Kit layout:

- `src/broadcast/recap.js`: remove `thread_ts`, add `buildRecapBlocks()` that
  renders a full-time card (header, flags+score, scorers, stage/group, divider,
  AI recap text). Fallback `text` includes both score and recap prose.
- `src/broadcast/scheduler.js`: drop the `threadTs` lookup for recap calls.
- Reuses `getFlag` from `src/broadcast/flags.js` (same as live card).
- Tests: verify Block Kit structure, no `thread_ts`, still posts when threadTs
  is null.

## Part 4: Match time everywhere (shared utility)

- New `utcTime(iso)` in `src/utils/format.js` -> fixed `"HH:MM UTC"` anchor.
  Slack's `<!date>` token localizes per viewer but has no timezone token and the
  server can't know each viewer's zone, so we pair the localized time with a
  shared UTC reference.
- schedule context line: `🕒 {slackDate} · {utcTime}`.
- `buildMatchBlocks` (`src/highlights/digest.js`): append the same
  `🕒 {slackDate} · {utcTime}` to each highlight card's meta line, omitted when
  the fixture has no date. Because `buildMatchBlocks` is shared, both the
  `/worldcup2026 highlights` command and the daily digest get it at once.
- Reuses one utility (`utcTime`) across schedule, broadcast, and highlights for
  a consistent format.
