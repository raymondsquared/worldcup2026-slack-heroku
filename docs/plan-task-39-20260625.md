# Task 39: Implement and Test All 5 MCP Tools

## Details

Date: 2026-06-25

Dependencies: task-38

Decisions: [adr-7, adr-8]

## Description

Implement 5 read-only MCP tools that wrap the Football API, enabling Heroku Managed
Inference Agents to access live World Cup 2026 data through standardized MCP protocol
for chat-based queries (@mentions, DMs).

Each tool will:

- Make direct HTTP calls to Football API (per ADR-8)
- Filter responses to World Cup 2026 fixtures only
- Use 10-second timeout per request
- Sanitize errors to prevent API key leaks
- Validate input parameters against JSON schema
- Return JSON responses compatible with MCP protocol

This enables always-fresh data for AI chat interactions while slash commands continue
to use the fast cached data layer (ADR-6).

## Acceptance Criteria

### Tool Implementation

- (Ubiquitous) The system shall implement `football_get_live_fixtures` tool that returns
  currently live World Cup 2026 fixtures
- (Ubiquitous) The system shall implement `football_get_fixtures_by_date` tool that accepts
  `date` parameter in YYYY-MM-DD format and returns fixtures for that date
- (Ubiquitous) The system shall implement `football_get_fixture_details` tool that accepts
  `fixture_id` number parameter and returns detailed fixture info including events, lineups,
  and statistics
- (Ubiquitous) The system shall implement `football_get_team_squad` tool that accepts `team_id`
  number parameter and returns team roster
- (Ubiquitous) The system shall implement `football_get_standings` tool that accepts optional
  `league_id` and `season` parameters and returns group standings (defaults to World Cup 2026)
- (Ubiquitous) All tools shall be prefixed with `football_` for namespace isolation
- (Ubiquitous) All tools shall be registered using the `registerTool()` function from
  `src/mcp/football-server.js`

### API Communication

- (Ubiquitous) All tools shall make HTTP requests directly to `https://v3.football.api-sports.io` using native fetch
- (Ubiquitous) All HTTP requests shall include `x-apisports-key` header with the API key from environment variable
- (Ubiquitous) All HTTP requests shall have a 10-second timeout using AbortSignal
- (Event-driven) When a request times out, the system shall return an error message indicating timeout
- (Event-driven) When the Football API returns 429 status, the system shall return a rate limit error message
- (Event-driven) When the Football API returns non-200 status, the system shall return a sanitized error message
- (Ubiquitous) All error messages shall be sanitized using `sanitizeError()` to replace API key with `[REDACTED]`

### Input Validation

- (Event-driven) When `football_get_fixtures_by_date` receives a date parameter, the system
  shall validate it matches YYYY-MM-DD format using regex
- (Event-driven) When `football_get_fixture_details` receives a fixture_id parameter, the
  system shall validate it is a number
- (Event-driven) When `football_get_team_squad` receives a team_id parameter, the system shall validate it is a number
- (Event-driven) When invalid parameters are provided, the system shall return a validation error with clear message
- (Ubiquitous) Parameter validation shall be defined in each tool's inputSchema using JSON Schema

### Response Filtering

- (Ubiquitous) All tools shall filter API responses to include only World Cup 2026 fixtures (league ID 1)
- (Ubiquitous) The system shall return filtered JSON responses compatible with MCP protocol specification

### Quality

- (Ubiquitous) All code shall pass `npm run lint`
- (Ubiquitous) All code shall follow existing ESLint and Prettier configuration
- (Ubiquitous) All tools shall have manual test procedures documented
- (Ubiquitous) The system shall be manually testable via MCP protocol stdin/stdout

## Implementation Plan

### Step 1: Create API Client Module (45 minutes)

1. Create `src/mcp/api-client.js` with shared HTTP request function:
   - Import native fetch
   - Define BASE_URL constant: `https://v3.football.api-sports.io`
   - Define TIMEOUT constant: 10000ms
   - Implement `makeRequest(endpoint, params)` function:
     - Build URL with query parameters
     - Add `x-apisports-key` header from `process.env.FOOTBALL_API_KEY`
     - Use `AbortSignal.timeout(10000)` for timeout
     - Catch timeout errors and return clear message
     - Catch 429 errors and return rate limit message
     - Catch non-200 responses and return status + body
     - Use `sanitizeError()` from football-server.js on all errors
     - Return parsed JSON on success

2. Define World Cup 2026 constant: `const WORLD_CUP_LEAGUE_ID = 1;`

3. Implement filtering helper:
   - `filterWorldCupFixtures(response)` - Filter API response to league ID 1 only

### Step 2: Implement football_get_live_fixtures Tool (30 minutes)

1. In `src/mcp/football-server.js`, add tool registration after `createServer()` function
2. Define tool schema:

```javascript
{
  name: 'football_get_live_fixtures',
  description: 'Get currently live World Cup 2026 fixtures with scores and events',
  inputSchema: {
    type: 'object',
    properties: {},
    required: []
  }
}
```

1. Implement handler:
   - Call `makeRequest('/fixtures', { live: 'all' })`
   - Filter response to World Cup 2026 fixtures
   - Return filtered JSON
1. Register tool: `registerTool(toolDef, handler)`

### Step 3: Implement football_get_fixtures_by_date Tool (30 minutes)

1. Define tool schema:

```javascript
{
  name: 'football_get_fixtures_by_date',
  description: 'Get World Cup 2026 fixtures for a specific date',
  inputSchema: {
    type: 'object',
    properties: {
      date: {
        type: 'string',
        description: 'Date in YYYY-MM-DD format',

    required: ['date']
  }
}
```

1. Implement handler:
   - Validate date parameter matches regex
   - Call `makeRequest('/fixtures', { date })`
   - Filter response to World Cup 2026 fixtures
   - Return filtered JSON
1. Register tool

## Test Strategy

### Manual Testing

- **Tool registration:** Send `tools/list` request via stdin, verify all 5 tools present
- **Live fixtures:** Call `football_get_live_fixtures`, verify returns live fixtures or empty
- **Date-based fixtures:** Call `football_get_fixtures_by_date` with valid date, verify returns fixtures
- **Invalid date:** Call with invalid date format, verify validation error
- **Fixture details:** Call `football_get_fixture_details` with fixture ID, verify returns events/lineups/stats
- **Team squad:** Call `football_get_team_squad` with team ID, verify returns roster
- **Standings:** Call `football_get_standings` with no params, verify returns World Cup 2026 standings
- **Standings with params:** Call with league_id and season, verify returns correct standings
- **Timeout:** Temporarily set low timeout, verify timeout error message
- **Rate limit:** Simulate 429 response (if possible), verify rate limit error handling
- **API key sanitization:** Trigger errors, verify no API key appears in any response
- **World Cup filtering:** Verify only league ID 1 fixtures returned from live/date endpoints

- Run `npm test` to ensure no regressions in existing test suite
- Run `npm run lint` to verify code style compliance
