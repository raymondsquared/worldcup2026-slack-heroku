# task-4: Static World Cup Data + Query Helpers

## Details

Date: 2026-06-15

Dependencies: task-1

Decisions: none

## Description

Add static JSON files with World Cup 2026 data (matches, teams, players) and
simple query helpers to look up schedules, groups, and upcoming matches. This
data powers slash commands (task-6), AI context (task-7), and broadcasts
(task-8).

Addresses TR-9 and FR-6.

## Acceptance Criteria

- The system shall include static JSON data for matches, teams, and players
  in `src/data/`
- The system shall export query helpers from `src/data/index.js`
- When `getUpcomingMatches(n)` is called, the system shall return the next n
  matches from now
- When `getGroups()` is called, the system shall return all groups with their
  teams
- When `getPlayers(teamCode)` is called, the system shall return players for
  that team
- When `npm test` is run, the system shall pass with tests covering the helpers
- When `npm run lint` is run, the system shall pass with no errors

## Implementation Plan

1. Create `src/data/matches.json` - array of match objects (date, teams,
   group stage/knockout)
2. Create `src/data/teams.json` - array of team objects (name, group, code)
3. Create `src/data/players.json` - array of player objects (name, team code,
   position, number)
4. Create `src/data/index.js` - exports: `getUpcomingMatches(n)`,
   `getGroups()`, `getAllMatches()`, `getPlayers(teamCode)`
5. Create `src/test/data.test.js` - unit tests for each helper
6. Run `npm test` and `npm run lint`

## Test Strategy

Run `npm test` and `npm run lint`.

- `getUpcomingMatches(3)` returns 3 matches after current date
- `getGroups()` returns correct group structure
- `getAllMatches()` returns full match list
- `getPlayers('USA')` returns players for that team
- JSON files are valid and non-empty
