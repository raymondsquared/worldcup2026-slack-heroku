# Task 40: Deploy and Document MCP Server

## Details

Date: 2026-06-25

Dependencies: task-39

Decisions: [adr-7, adr-8]

## Description

Deploy the Football API MCP server to Heroku Common Runtime and integrate with Heroku
Managed Inference via the `/v1/agents/heroku` endpoint. Add comprehensive documentation
including example tool calls, curl commands for testing, and integration instructions.

This task ensures the MCP server is accessible to Heroku Managed Inference for AI chat
interactions. It validates that all 5 tools are discoverable and callable via the
inference API, and provides documentation for future maintenance and debugging.

Critical Constraint: MCP servers must run on Common Runtime (public). Private Spaces
do not support MCP integration with Heroku Managed Inference.

## Acceptance Criteria

### Deployment

- (Ubiquitous) The MCP server shall be deployed to Heroku as the `mcp-football` process
  type
- (Ubiquitous) The Procfile shall contain the entry:
  `mcp-football: node src/mcp/football-server.js`
- (Ubiquitous) The FOOTBALL_API_KEY environment variable shall be configured in Heroku
  config vars
- (Event-driven) When the app is deployed, the system shall start the mcp-football
  process successfully
- (Event-driven) When checking Heroku logs, the system shall show "Football API MCP
  server running on stdio" message

### MCP Registration

- (Ubiquitous) The MCP server shall be deployed to Common Runtime (not Private Space)
- (Ubiquitous) All 5 tools shall be registered and callable via `/v1/agents/heroku`
- (Ubiquitous) Tools must be specified individually by name, not by namespace
- (Event-driven) When calling a tool via the agent endpoint, the system shall execute
  successfully with < 2s response time (p95)

### Documentation

- (Ubiquitous) The README.md shall document the MCP server deployment and usage
- (Ubiquitous) Documentation shall include example curl commands for testing tools via
  inference API
- (Ubiquitous) Documentation shall include example MCP protocol requests for direct
  testing
- (Ubiquitous) Documentation shall list all 5 tools with their parameters and purpose
- (Ubiquitous) Documentation shall explain the difference between MCP tools (chat) and
  ADR-6 tools (slash commands)

### Verification

- (Ubiquitous) At least one tool shall be tested via the Heroku Inference API to verify
  end-to-end functionality
- (Ubiquitous) The test shall confirm zero API key leaks in responses
- (Ubiquitous) The test shall verify World Cup 2026 filtering is applied
- (Ubiquitous) Tool call error rate shall be < 5% under normal conditions

### Step 4: Test Tools via Inference API (30 minutes)

1. Test football_get_live_fixtures via inference API:

   ```bash
   curl -X POST "$INFERENCE_URL/v1/agents/heroku" \
     -H "Authorization: Bearer $INFERENCE_KEY" \
     -H "Content-Type: application/json" \
     -H "X-Forwarded-Proto: https" \
     -d '{
       "model": "'"$INFERENCE_MODEL_ID"'",
       "messages": [{"role": "user", "content": "What World Cup matches are live right now?"}],
       "tools": [
         {"type": "mcp", "name": "football_get_live_fixtures"},
         {"type": "mcp", "name": "football_get_standings"}
       ]
     }'
   ```

2. Verify response contains:
   - Tool call to football_get_live_fixtures
   - Valid JSON response
   - No API key leaks
   - Only World Cup 2026 fixtures (league ID 1)

3. Test error handling with invalid date:

   ```bash
   curl -X POST "$INFERENCE_URL/v1/agents/heroku" \
     -H "Authorization: Bearer $INFERENCE_KEY" \
     -H "Content-Type: application/json" \
     -H "X-Forwarded-Proto: https" \
     -d '{
       "model": "'"$INFERENCE_MODEL_ID"'",
       "messages": [{"role": "user", "content": "Show me World Cup fixtures for date invalid-format"}],
       "tools": [
         {"type": "mcp", "name": "football_get_fixtures_by_date"}
       ]
     }'
   ```

4. Verify error response:
   - Returns validation error
   - No API key in error message

5. Measure response time:
   - Should be < 2s for p95
   - Note: STDIO transport adds ~300-500ms cold start (per ADR-7)

### Step 5: Create Documentation (1 hour)

1. Add MCP Server section to README.md after Data Sync section:

   ```markdown
   ## MCP Server (Football API)
   
   The app includes an MCP server that exposes Football API data to Heroku Managed
   Inference for AI chat interactions.
   
   ### Deployment
   
   The MCP server is automatically deployed with the app via the `mcp-football`
   process type in Procfile. It uses STDIO transport and runs on-demand (one-off
   dynos) when called by Heroku Inference.
   
   ### Available Tools
   
   1. football_get_live_fixtures - Get currently live World Cup 2026 fixtures
   2. football_get_fixtures_by_date - Get fixtures for a specific date (YYYY-MM-DD)
   3. football_get_fixture_details - Get detailed fixture with events/lineups/stats
   4. football_get_team_squad - Get team roster for a team
   5. football_get_standings - Get group standings (defaults to World Cup 2026)
   
   ### Usage
   
   Via Heroku Managed Inference (AI Chat):
   
   When users @mention the bot or send DMs, MCP tools are automatically called to
   fetch fresh World Cup data.
   
   Direct Testing via MCP Protocol:
   
   ```bash
   # Test locally with stdin/stdout
   echo '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}' | \
     node --env-file=.env src/mcp/football-server.js 2>/dev/null
   
   # Or use the automated test suite
   npm run test:mcp
   ```

   Testing via Heroku Inference API:

   ```bash
   curl -X POST "$INFERENCE_URL/v1/agents/heroku" \
     -H "Authorization: Bearer $INFERENCE_KEY" \
     -H "Content-Type: application/json" \
     -H "X-Forwarded-Proto: https" \
     -d '{
       "model": "'"$INFERENCE_MODEL_ID"'",
       "messages": [{"role": "user", "content": "What matches are live?"}],
       "tools": [
         {"type": "mcp", "name": "football_get_live_fixtures"},
         {"type": "mcp", "name": "football_get_standings"}
       ]
     }'
   ```

   ### Architecture

   - MCP Tools vs ADR-6 Tools: MCP tools (AI chat) fetch fresh data directly from
     Football API. ADR-6 tools (slash commands) use cached data layer for speed.
   - Transport: STDIO (one-off dynos, ~300-500ms cold start)
   - Security: All errors sanitized to prevent API key leaks
   - Filtering: Responses filtered to World Cup 2026 only (league ID 1)

   See ADR-7 (STDIO transport) and ADR-8 (direct API calls) for architectural decisions.
