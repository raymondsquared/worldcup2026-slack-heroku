# Task 41: Chart Generation Utility with Tests

## Details

Date: 2026-06-25

Dependencies: task-40

Decisions: none

## Description

Create a chart generation utility that produces QuickChart URLs for ball possession
(donut chart) and total shots (horizontal bar chart). These chart images will be
embedded in post-match recap cards via Slack Block Kit image blocks.

## Acceptance Criteria

- Ball possession donut chart generates valid QuickChart URL with correct percentages
- Total shots horizontal bar chart generates valid QuickChart URL with correct counts
- Charts use team colors or distinct colors for home/away
- Functions return null when statistics data is missing (graceful skip)
- Unit tests cover: valid data, missing data, edge cases (0%, 100%, null values)
- `npm test` and `npm run lint` pass

## Data Shape

Statistics come from `liveData.statistics` (array of team stat objects):

```json
[
  {
    "teamId": "ECU",
    "statistics": [
      { "type": "Ball Possession", "value": "75%" },
      { "type": "Total Shots", "value": 27 }
    ]
  },
  {
    "teamId": "CUW",
    "statistics": [
      { "type": "Ball Possession", "value": "25%" },
      { "type": "Total Shots", "value": 5 }
    ]
  }
]
```

Key data facts:

- `Ball Possession` value is a string with `%` suffix (e.g. `"75%"`)
- `Total Shots` value is a number (e.g. `27`)
- Either stat can be `null` or missing entirely
- `teamId` can be a string (FIFA code) or number (API ID)

## Implementation Plan

### Step 1: Create `src/broadcast/charts.js`

Two exported functions:

```javascript
// Returns QuickChart URL string or null if data missing
function possessionChartUrl(statistics, homeTeamId, awayTeamId, homeTeamName, awayTeamName)
function shotsChartUrl(statistics, homeTeamId, awayTeamId, homeTeamName, awayTeamName)
```

The caller passes team IDs so the function can match the correct stats entry to
home/away (the `statistics` array order is not guaranteed to be home-first).

Possession donut chart:

- Type: doughnut
- Parse `"75%"` → `75` (parseInt)
- Match stats to home/away by `teamId`
- Data order: away first so home (blue) renders on left side of doughnut
- Legend: reversed so it reads `[blue] Home [pink] Away` left-to-right
- All text 5px (legend, title, datalabels)
- Text color: medium gray (#999)
- Border: none (borderWidth:0)
- Colors: home light blue (#90CAF9), away light pink (#F48FB1)
- Size: 160x115

Match Stats horizontal bar chart:

- Type: horizontalBar
- Data rows: Total Shots (required), Shots on Goal, Pass %, Fouls (optional)
- Each team is a dataset (home vs away as grouped bars)
- Legend always shown (team names)
- All text 5px (legend, title, axis ticks, datalabels)
- Text color: medium gray (#999)
- Rounded bars (borderRadius:5), no border
- Colors: home light blue (#90CAF9), away light pink (#F48FB1)
- Size: 195x115

QuickChart URL format:

```
https://quickchart.io/chart?c={encoded_config}&w=160&h=115&bkg=white
```

### Step 2: Create `src/test/broadcast-charts.test.js`

Test cases:

1. Valid possession data → returns URL containing correct percentages
2. Valid shots data → returns URL containing correct shot counts
3. Missing statistics array → returns null
4. Empty statistics array → returns null
5. Possession stat is null → returns null
6. Total Shots stat is null → returns null
7. Edge case: 0% vs 100% → still generates valid URL
8. Edge case: both teams 50% → generates valid URL
9. URL is properly encoded (no invalid chars)
10. Team names appear in chart labels

### Step 3: Verify

```bash
npm test
npm run lint
```

## Notes

- QuickChart.io is free, no API key needed, HTTPS URLs work in Slack
- Chart config uses JS string template format (not JSON.stringify) to preserve
  formatter functions like `(v)=>v+'%'`
- Transparent background (`bkg=transparent`) so charts blend with Slack themes
- Keep chart configs minimal — URL limit is ~16k chars but shorter is better
- Slack image blocks display at fixed widths, don't over-size charts
- No doughnut center-text plugin (adds complexity, not worth it for demo)
- `statistics` array order varies — always match by `teamId`, never assume index
- Team names with apostrophes escaped via `esc()` helper (e.g. Côte d'Ivoire)
- Charts integrated in daily digest and `/highlights` (task-42), NOT post-match
  recap (stats are empty at full-time)
