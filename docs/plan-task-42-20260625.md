# Task 42: Integrate Charts into Daily Digest & Highlights

## Details

Date: 2026-06-25

Dependencies: task-41

Decisions:

- Charts go in the daily digest and `/highlights` command, NOT the post-match recap.
  Reason: at full-time the API hasn't populated statistics yet (live cache is empty).
  Statistics only become available after `npm run sync` pulls them into fixture files.

## Description

Integrate the chart generation utility (task-41) into the daily highlights digest
and the `/worldcup2026 highlights` slash command. Both surfaces read from synced
fixture JSON files which contain statistics after `npm run sync`.

## Acceptance Criteria

- Possession chart image appears below each match in digest when stats available
- Shots chart image appears below each match in digest when stats available
- Charts are gracefully skipped when stats are missing (no error, no empty block)
- Existing digest/highlights block structure preserved
- Existing tests still pass, new tests cover chart integration
- `npm test` and `npm run lint` pass

## Block Layout (daily digest / highlights, per match)

```text
section:   🇲🇽 *Mexico*  `2 - 0`  *South Africa* 🇿🇦  [▶️ Watch]
image:     Ball Possession (doughnut)     <- NEW
image:     Total Shots (horizontal bar)   <- NEW
```

Charts only appear if the respective stat exists in the synced fixture file.

## Implementation Plan

### Step 1: Modify `src/highlights/digest.js`

1. Import chart functions:

   ```javascript
   const { possessionChartUrl, shotsChartUrl } = require('../broadcast/charts');
   ```

2. In `buildMatchBlocks()`, after the match section block, add chart image blocks:

   ```javascript
   const stats = fixture?.statistics;
   if (stats) {
     const possUrl = possessionChartUrl(stats, homeId, awayId, home, away);
     if (possUrl) {
       blocks.push({ type: 'image', image_url: possUrl, alt_text: 'Ball Possession' });
     }
     const shotsUrl = shotsChartUrl(stats, homeId, awayId, home, away);
     if (shotsUrl) {
       blocks.push({ type: 'image', image_url: shotsUrl, alt_text: 'Total Shots' });
     }
   }
   ```

### Step 2: Update tests

- `src/test/highlights-digest.test.js`: add tests for chart blocks present/absent
- `src/test/commands.test.js`: update block count assertion for highlights

### Step 3: Verify

```bash
npm test
npm run lint
```

## Notes

- Slack `image` blocks require `image_url` (HTTPS) and `alt_text`
- QuickChart URLs are HTTPS, no auth needed - Slack can render directly
- Statistics come from synced fixture files (`src/data/fixtures/*.json`), loaded
  via `getFixtureById()` - same data path as highlight video URLs
- Post-match recap (`recap.js`) does NOT include charts because the live cache
  has no statistics at full-time
