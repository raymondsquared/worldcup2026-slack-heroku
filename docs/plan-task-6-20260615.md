# task-6: Slash Commands

## Details

Date: 2026-06-15

Dependencies: task-2, task-4

Decisions: none

## Description

Register the `/worldcup2026` slash command with two subcommands: `schedule`
and `groups`. Uses Block Kit for formatted responses. Data comes from the
static data layer (task-4).

Addresses FR-1, FR-2, FR-3.

## Acceptance Criteria

- The app shall register a `/worldcup2026` slash command handler
- `/worldcup2026 schedule` shall respond with the next 3 upcoming matches
  formatted as Block Kit blocks (date/time, team names)
- `/worldcup2026 groups` shall respond with all groups and their teams
  formatted as Block Kit blocks
- Unknown or missing subcommands shall respond with help text listing
  available subcommands
- When `npm test` is run, the system shall pass with tests covering both
  subcommands and the help fallback
- When `npm run lint` is run, the system shall pass with no errors

## Implementation Plan

1. Add `getTeamName(id)` helper to `src/data/index.js` - returns team name
   from ID (e.g., `"USA"` -> `"United States"`)

2. Create `src/commands/worldcup.js` - slash command handler:
   - Export a `register(app)` function that calls `app.command()`
   - Parse `command.text` to determine subcommand (`schedule`, `groups`)
   - `schedule` - calls `getUpcomingMatches(3)`, resolves team IDs to names
     via `getTeamName()`, builds Block Kit sections with date/time
     (formatted readable string from `dateAndTimeInUTC`) and team names
   - `groups` - calls `getGroups()`, builds Block Kit sections with group
     name and team names
   - Fallback - responds with usage help text
   - Calls `ack()` immediately before any processing

3. Wire up in `src/app.js` - import and call `register(app)`

4. Create `src/test/commands.test.js` - unit tests:
   - Mock `ack` and `respond` from Bolt
   - Test `ack()` is always called first
   - Test `schedule` returns blocks with team names and formatted date/time
   - Test `groups` returns blocks with group data
   - Test empty/unknown subcommand returns help text

5. Run `npm test` and `npm run lint`

## Test Strategy

Run `npm test` and `npm run lint`.

- Mock Bolt's `ack()` and `respond()` functions
- `schedule` - verify respond called with blocks containing team names (not
  IDs) and formatted date/time
- `groups` - verify respond called with blocks containing group names and
  team names
- Help fallback - verify respond called with usage text for empty or unknown
  subcommands
