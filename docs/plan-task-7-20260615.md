# task-7: AI Chat Handlers

## Details

Date: 2026-06-15

Dependencies: task-2, task-5

Decisions: none

## Description

Handle @mentions and DMs by passing the user's message through the MIA
guardrail pipeline (`ask()`) and replying with AI-generated responses
grounded in World Cup data. Mentions reply in-thread; DMs reply directly.

Addresses FR-4, FR-5, FR-6.

## Acceptance Criteria

- When a user @mentions the bot in a channel, the app shall respond in the
  same thread with an AI-generated answer
- When a user sends a DM to the bot, the app shall respond conversationally
- The AI response shall be grounded in World Cup data via the existing
  `ask()` pipeline (mask -> sanitize -> filterToxic -> ground -> MIA ->
  filterToxic -> demask -> log)
- Errors from MIA shall be caught and the user shall receive a friendly
  error message
- When `npm test` is run, the system shall pass with tests covering mention
  and DM handlers
- When `npm run lint` is run, the system shall pass with no errors

## Implementation Plan

1. Create `src/handlers/chat.js` - event handlers:
   - Export a `register(app)` function
   - `app.event('app_mention')` - extract user text (strip `<@BOT_ID>`),
     call `ask(text)`, reply in thread via
     `say({ thread_ts: event.thread_ts || event.ts, text })`
   - `app.message()` - guard: skip if `message.bot_id` (prevent loop),
     skip if `message.channel_type !== 'im'` (DMs only),
     call `ask(text)`, reply via `say(text)`
   - Both handlers wrap `ask()` in try/catch, respond with friendly error
     on failure

2. Wire up in `src/app.js` - import and call `register(app)`

3. Create `src/test/chat.test.js` - unit tests:
   - Mock `say` from Bolt
   - Mock `ask()` from `src/mia`
   - Test mention handler replies in thread (passes `thread_ts`)
   - Test mention handler strips bot ID from message text
   - Test DM handler replies directly
   - Test DM handler ignores bot messages (no loop)
   - Test DM handler ignores non-IM channel messages
   - Test error from `ask()` returns friendly error message

4. Run `npm test` and `npm run lint`

## Test Strategy

Run `npm test` and `npm run lint`.

- Mock `say()` and `require('../mia').ask`
- Mention: verify `say` called with `thread_ts` matching event `ts`, and
  response text from `ask()`
- DM: verify `say` called with text from `ask()`, no `thread_ts`
- Error: mock `ask()` to throw, verify `say` called with error message
- Strip mention: input `<@U123> when does USA play?` -> `ask` receives
  `when does USA play?`
- Bot loop: message with `bot_id` set -> `say` never called
- Channel filter: message with `channel_type !== 'im'` -> `say` never called
