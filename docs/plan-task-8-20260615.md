# task-8: Match Broadcasting

## Details

Date: 2026-06-15

Dependencies: task-2, task-4

Decisions: none

## Description

Post match cards to a configured Slack channel approximately 15 minutes
before each match's scheduled kickoff. A simple interval-based scheduler
checks for upcoming matches and posts Block Kit cards with team names and
kickoff time. Uses `setInterval` - no external scheduler dependency.

Addresses FR-7, FR-8, TR-11.

## Acceptance Criteria

- The app shall check for upcoming matches at a regular interval
- When a match is approximately 15 minutes away, the app shall post a Block
  Kit match card to the channel specified by `BROADCAST_CHANNEL_ID` env var
- Match cards shall include team names (resolved from IDs) and kickoff time
  (Slack native date format for user-local timezone)
- The system shall not post duplicate cards for the same match
- The scheduler shall be startable and stoppable (for testing)
- When `npm test` is run, the system shall pass with tests covering the
  scheduler and match card formatting
- When `npm run lint` is run, the system shall pass with no errors

## Implementation Plan

1. Extract `slackDate(isoString)` from `src/commands/worldcup.js` into
   `src/utils/format.js` - shared helper for Slack native date formatting.
   Update `commands/worldcup.js` to import from the new location.

2. Add `BROADCAST_CHANNEL_ID` to `REQUIRED_ENV_VARS` in `src/index.js`

3. Create `src/broadcast/scheduler.js` - interval-based scheduler:
   - `start(client)` - accepts Bolt `client` (web API), starts
     `setInterval` checking every 60 seconds
   - Each tick: call `getUpcomingMatches(3)`, find matches where
     `0 < (kickoff - now) <= 15 minutes` (excludes already-started matches),
     post card for any not already posted
   - Track posted match IDs in a `Set` (in-memory, resets on restart - fine
     for demo per C-1). Only add to Set after successful `postMessage`.
   - On `postMessage` failure: log error, do NOT add match ID to Set
     (allows retry next tick)
   - `stop()` - clears the interval (for testing and graceful shutdown)
   - `buildMatchCard(match)` - Block Kit blocks with team names and Slack
     date format (via shared `slackDate`)

4. Wire up in `src/index.js` - after `app.start()`, call
   `scheduler.start(app.client)`

5. Create `src/test/broadcast.test.js` - unit tests:
   - Mock `client.chat.postMessage`
   - Test `buildMatchCard` returns Block Kit with team names and date
   - Test scheduler posts card for match within 15-minute window
   - Test scheduler skips already-posted matches (no duplicates)
   - Test scheduler skips matches outside the 15-minute window
   - Test `stop()` clears the interval

6. Run `npm test` and `npm run lint`

## Test Strategy

Run `npm test` and `npm run lint`.

- Mock `client.chat.postMessage` to capture posted messages
- Use fake timers (`jest.useFakeTimers`) to control interval ticks
- Inject a reference date or mock `Date.now()` for deterministic window
  checks
- `buildMatchCard` - verify blocks contain team names (not IDs) and Slack
  `<!date^...>` syntax
- Duplicate prevention - call tick twice for same match, verify only one
  post
- Window check - match 20 min away -> no post; match 10 min away -> post;
  match already started -> no post
- Error handling - mock `postMessage` to throw, verify match not marked as
  posted (retry on next tick)
