## Description

HTTP client module for the football API integration. Foundation for all live data features -
both the sync script (task-10) and the live data poller (task-11) depend on this client. Uses
native `fetch` (Node 22 built-in) with no new dependencies, following the same pattern as
the existing `src/mia/client.js`.

The client authenticates via `x-apisports-key` header using the `FOOTBALL_API_KEY` env var,
provides endpoint helpers, and implements error handling that never exposes the API key in logs.

## Status: DONE

## Actual Implementation

File: `src/live-data/api.js`

- Base URL: `FOOTBALL_API_URL` env var
- `FOOTBALL_API_KEY` read from env at module load (throws if missing)
- 10s timeout via `AbortSignal.timeout(TIMEOUT_IN_MS)`
- `sanitizeError(message)` - replaces API key with `[REDACTED]` (TR-5)
- Internal `request(endpoint, params)` helper:
  - Builds URL with query params
  - Sets header: `x-apisports-key`
  - On non-2xx: throws with status + body (key sanitized)
  - On timeout: throws descriptive timeout error
  - Returns `response.json()`
- Exported functions:
  - `fetchLiveFixtures()` - `/fixtures?live=all`
  - `fetchFixturesByDate(date)` - `/fixtures?date=${date}`
  - `fetchFixtureById(fixtureId)` - `/fixtures?id=${id}` (full detail with events/statistics)
  - `fetchSquad(teamId)` - `/players/squads?team=${id}`
  - `fetchStandings(leagueId, season)` - `/standings?league=${id}&season=${year}`

Tests: `src/test/football-api/football-api.test.js` with mocked `global.fetch`.
