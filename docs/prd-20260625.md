# PRD: Football API MCP Server

Date: 2026-06-25

This PRD covers two features for World Cup 2026 Slack bot iteration 6:

1. Football API MCP Server - Deploy an MCP server to Heroku that wraps the
   Football API, enabling Heroku Managed Inference Agents to access live World
   Cup 2026 data through standardized MCP tools for chat-based queries.

2. Match Recap Charts - Add visual statistics to post-match recap cards:
   ball possession donut chart and total shots horizontal bar chart.

Decisions: [adr-7, adr-8]

## Background and Strategic Fit

### Feature 1: Football API MCP Server

The Heroku Managed Inference Agents (MIA) service powers AI-driven interactions
in the World Cup 2026 Slack bot. While the bot has access to static data files
and an internal live cache through client-side tool calling (ADR-6), there is
value in exposing Football API data through a standardized MCP server interface
that follows Heroku's recommended patterns for MCP integration.

This MCP server will provide:

1. Standardized MCP protocol - Following Heroku's `mcp-code-exec-node` pattern
   for consistent MCP server deployment
2. Direct API access - Tools that fetch fresh data directly from Football API
   without local caching layers
3. External client support - Enabling future integrations beyond the current
   MIA implementation
4. Learning implementation - Demonstrating Heroku MCP best practices for
   future MCP servers (highlights, web search, etc.)

### Feature 2: Match Recap Charts

Post-match recap cards currently display AI-generated text summaries with score
lines and goal scorers. Adding visual statistics (ball possession, total shots)
enhances engagement and provides quick visual insight into match dynamics without
requiring users to read detailed statistics.

Both features align with the project's goal of leveraging Heroku Managed Inference
capabilities and improving the World Cup 2026 Slack bot user experience.

## Success Criteria

Feature 1: Football API MCP Server

- MCP server successfully registered and discoverable via
  `/v1/mcp/servers` API endpoint
- All 5 tools callable through Heroku Managed Inference with < 2s response
  time (p95)
- Zero API key leaks in logs, error messages, or responses
- Tool call error rate < 5% under normal Football API availability
- MCP server deploys successfully as part of existing Heroku app
- Documentation complete with example tool calls and curl commands

Feature 2: Match Recap Charts

- Ball possession donut chart displays correct percentages for both teams
- Total shots horizontal bar chart displays correct shot counts for both teams
- Charts render as images in Slack recap card blocks
- Charts display gracefully when statistics data is unavailable (show "N/A" or skip)
- Chart generation adds < 500ms to recap posting latency

## Scope

What is included in this project?

Feature 1: Football API MCP Server

- MCP server implementation in Node.js using `@modelcontextprotocol/sdk`
- 5 read-only MCP tools wrapping Football API endpoints
- MCP tools used for chat-based AI queries (@mentions, DMs) to ensure fresh data
- STDIO transport mode (Heroku recommended pattern)
- Deployment as additional Procfile process on existing Heroku app
- Heroku Inference add-on attachment and MCP server registration
- Error handling with API key sanitization
- Basic testing via MCP Inspector and curl
- Documentation in README and example tool calls

Feature 2: Match Recap Charts

- Ball possession donut chart generation (using QuickChart or similar API)
- Total shots horizontal bar chart generation
- Chart image URLs embedded in Slack Block Kit image blocks
- Chart rendering in `src/broadcast/recap.js` buildRecapBlocks function
- Graceful fallback when statistics data is missing
- Unit tests for chart URL generation

## Out of Scope

What are we explicitly not doing in this version?

Feature 1: Football API MCP Server

- Write operations to Football API (read-only only)
- Response caching layer (call Football API directly for chat freshness)
- Batch operations (one fixture/team per tool call)
- Rate limiting logic (rely on Football API limits)
- Integration with existing trust-layer guardrails (separate from ADR-6 tools)
- Modifying slash commands (continue using ADR-6 data layer tools for fast cached responses)
- Modifying poller or sync scripts (continue using `src/live-data/api.js` unchanged)
- Additional MCP servers (highlights, web search - separate projects)
- SSE/HTTP transport modes (STDIO only)
- Local development MCP server setup (Heroku deployment only)

Feature 2: Match Recap Charts

- Self-hosted chart generation (use external chart API service)
- Interactive charts (static images only)
- Additional chart types beyond possession donut and shots bar chart
- Historical chart comparisons (single match only)
- Chart customization options (use default styling)

## Requirements

### Functional Requirements

Feature 1: Football API MCP Server

- FR-1: Expose 5 read-only tools via MCP protocol:
  - FR-1.1: `football_get_live_fixtures` - Get currently live World Cup fixtures
  - FR-1.2: `football_get_fixtures_by_date` - Get fixtures for specific date
    (requires `date` parameter in YYYY-MM-DD format)
  - FR-1.3: `football_get_fixture_details` - Get detailed fixture info including
    events, lineups, statistics (requires `fixture_id` number parameter)
  - FR-1.4: `football_get_team_squad` - Get team roster (requires `team_id`
    number parameter)
  - FR-1.5: `football_get_standings` - Get group standings (optional `league_id`
    and `season` parameters, default to World Cup 2026)
- FR-2: Deploy as `mcp-football` Procfile process on existing Heroku app
- FR-3: Register with Heroku Managed Inference endpoint automatically
- FR-4: Return JSON responses compatible with MCP protocol specification
- FR-5: Sanitize all error messages to prevent API key exposure
- FR-6: Support tool parameter validation per JSON schema (date format regex,
  numeric ID validation)
- FR-7: Filter API responses to World Cup 2026 fixtures only

Feature 2: Match Recap Charts

- FR-8: Generate ball possession donut chart showing percentage split between teams
- FR-9: Generate total shots horizontal bar chart showing shot counts for both teams
- FR-10: Embed chart images in Slack Block Kit image blocks within recap card
- FR-11: Extract ball possession and total shots from liveData.statistics
- FR-12: Handle missing statistics gracefully (skip charts or show placeholder)

### Technical Requirements

Feature 1: Football API MCP Server

- TR-1: Use `@modelcontextprotocol/sdk` for STDIO transport implementation
- TR-2: Make HTTP requests directly to Football API using fetch with patterns
  similar to `src/live-data/api.js` (independent implementation, not code reuse)
- TR-3: 10-second timeout on all Football API HTTP requests
- TR-4: Follow existing code style (ESLint, Prettier configuration)
- TR-5: Pass `npm test` and `npm run lint` before deployment
- TR-6: Run on Node.js 22+ (existing engine requirement)
- TR-7: Tool names prefixed with `football_` for namespace isolation
- TR-8: Implement API key sanitization by replacing API key values with
  `[REDACTED]` in all error messages and logs

Feature 2: Match Recap Charts

- TR-9: Use QuickChart API (<https://quickchart.io>) or similar free chart service
- TR-10: Generate chart URLs with properly URL-encoded parameters
- TR-11: Chart images must be accessible via HTTPS for Slack display
- TR-12: Implement chart generation in separate utility module (e.g., `src/broadcast/charts.js`)
- TR-13: Add unit tests for chart URL generation with sample statistics data

### Constraints

Feature 1: Football API MCP Server

- Must not exceed Football API rate limits (plan-dependent)
- Must run in Heroku's STDIO execution model (one-off dynos)
- Must follow AGENTS.md security guardrails (Zero Data Retention,
  API key sanitization)
- Cannot modify existing ADR-6 tool calling implementation
- Must coexist with existing Slack bot worker process

Feature 2: Match Recap Charts

- Chart service must be free tier or have generous limits (no additional costs)
- Chart generation must not block recap posting (use async or timeout)
- Must work within Slack's image block constraints (HTTPS URLs, reasonable size)

## Assumptions

Feature 1: Football API MCP Server

- Heroku Inference add-on will be attached to app before deployment
- `FOOTBALL_API_KEY` is already configured in Heroku config vars
- `INFERENCE_URL` and `INFERENCE_KEY` are already configured
- Football API rate limits sufficient for expected MCP tool usage
- MCP protocol is stable (no breaking changes during development)
- Heroku's MCP registration happens automatically via Procfile declaration
- One-off dyno execution model is acceptable for tool latency

Feature 2: Match Recap Charts

- QuickChart or similar service has sufficient free tier limits for World Cup matches
  (max ~64 matches, ~3 recap cards per day during tournament)
- Statistics data (ball possession, total shots) is consistently available from Football API
- Slack reliably displays HTTPS image URLs in image blocks
- Chart service has acceptable uptime (>99%); fallback to text-only recap acceptable
